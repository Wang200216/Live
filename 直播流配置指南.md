## 🎯 配置真实直播的三种方式

### 方式一：使用云直播服务（推荐）⭐

这是最常用和稳定的方式，适合生产环境。

#### 1. 选择云服务商

**推荐的云服务商**：
- **腾讯云直播**（推荐，性价比高）
- **阿里云直播**（稳定，大公司）
- **七牛云直播**（便宜，适合小项目）
- **网易云信**（专注音视频）

#### 2. 腾讯云直播配置步骤

**步骤1：开通服务**
```
1. 注册腾讯云账号：https://cloud.tencent.com
2. 进入"云直播"产品页
3. 开通服务（实名认证）
4. 购买流量包（可选，按需购买）
```

**步骤2：添加域名**
```
1. 进入云直播控制台
2. 添加推流域名（例如：push.yourdomain.com）
3. 添加播放域名（例如：play.yourdomain.com）
4. 配置CNAME解析
```

**步骤3：获取推流地址**
```
推流地址格式：
rtmp://push.yourdomain.com/live/streamname?txSecret=xxx&txTime=xxx

关键参数：
- live: 应用名称（AppName）
- streamname: 流名称（StreamName）
- txSecret: 防盗链签名
- txTime: 过期时间戳
```

**步骤4：获取播放地址**
```
RTMP播放: rtmp://play.yourdomain.com/live/streamname
HLS播放:  https://play.yourdomain.com/live/streamname.m3u8  ✅ 推荐用这个
FLV播放:  https://play.yourdomain.com/live/streamname.flv
```

**步骤5：配置到小程序**

修改 `config/live-config.js` 文件：
```javascript
export default {
  // 将播放地址替换为你的HLS地址
  liveStreamUrl: 'https://play.yourdomain.com/live/streamname.m3u8',
  
  config: {
    autoplay: true,
    muted: false,
    mode: 'live',
    objectFit: 'contain',
    minCache: 1,
    maxCache: 3
  }
}
```

**步骤6：开始推流**

使用OBS或其他推流软件：
```
1. 下载OBS Studio: https://obsproject.com
2. 设置 → 推流
3. 服务：自定义
4. 服务器：rtmp://push.yourdomain.com/live
5. 串流密钥：streamname?txSecret=xxx&txTime=xxx
6. 点击"开始推流"
```

### 方式三：自建直播服务器（高级）

#### 使用SRS（Simple Realtime Server）

**适用场景**：
- 想要完全控制
- 有服务器资源
- 技术能力强

**快速部署**：

```bash
# 1. 使用Docker部署SRS
docker run -d --name srs \
  -p 1935:1935 \
  -p 1985:1985 \
  -p 8080:8080 \
  registry.cn-hangzhou.aliyuncs.com/ossrs/srs:5

# 2. 推流地址
rtmp://你的服务器IP:1935/live/livestream

# 3. 播放地址（需要配置HLS）
http://你的服务器IP:8080/live/livestream.m3u8
```

**详细配置**：
```bash
# 1. 创建配置文件 srs.conf
listen              1935;
max_connections     1000;
daemon              off;
pid                 /var/run/srs.pid;

http_server {
    enabled         on;
    listen          8080;
}

vhost __defaultVhost__ {
    hls {
        enabled         on;
        hls_path        /tmp/hls;
        hls_fragment    10;
        hls_window      60;
    }
}

# 2. 启动SRS
docker run -d --name srs \
  -p 1935:1935 \
  -p 8080:8080 \
  -v /path/to/srs.conf:/usr/local/srs/conf/srs.conf \
  registry.cn-hangzhou.aliyuncs.com/ossrs/srs:5
```

后端需要支持以下接口来动态配置直播流：

**接口**: `POST /api/admin/live/start`

```json

  "streamUrl": "https://play.yourdomain.com/live/main.m3u8",
```

- 如果传了 `streamUrl`，直接使用该地址
- 后端返回的 `streamUrl` 会推送给小程序

## ⚙️ 不同协议说明

### HLS (HTTP Live Streaming) ✅ 推荐

**格式**：`https://your-domain.com/live/stream.m3u8`

**优点**：
- ✅ 支持HTTPS，安全
- ✅ 兼容性最好
- ✅ 适合移动端
- ✅ 微信小程序原生支持

**缺点**：
- ⚠️ 延迟较高（10-30秒）

**使用场景**：
- 微信小程序直播（必须用这个）
- 移动端直播
- 观看人数多的场景

### RTMP (Real Time Messaging Protocol)

**格式**：`rtmp://your-domain.com/live/stream`

**优点**：
- ✅ 延迟低（1-3秒）
- ✅ 稳定性好

**缺点**：
- ❌ 不支持HTTPS
- ❌ 微信小程序不支持
- ❌ 浏览器需要Flash（已淘汰）

**使用场景**：
- 推流（OBS推流到服务器）
- 服务器间传输

### FLV (Flash Video)

**格式**：`https://your-domain.com/live/stream.flv`

**优点**：
- ✅ 延迟较低（3-5秒）
- ✅ 支持HTTPS

**缺点**：
- ⚠️ 兼容性一般
- ⚠️ 微信小程序支持有限

**使用场景**：
- PC端H5直播
- 特定场景使用

## 🌐 微信小程序直播限制

### 必须使用的协议

✅ **支持的协议**：
- HLS (m3u8) ✅ 推荐

- WebRTC

**小程序上线时需要**：
- ✅ 必须使用HTTPS（m3u8）

**开发测试时**：
- ✅ 可以使用HTTP（开发工具中勾选"不校验合法域名"）

## 🔍 常见问题

### Q1：小程序中直播画面是黑屏？

**检查清单**：
1. ✅ 确认直播流地址可以在VLC中播放
2. ✅ 确认使用的是HLS (m3u8) 格式
3. ✅ 确认网络可以访问该地址
4. ✅ 查看小程序控制台是否有错误
5. ✅ 确认已经开始推流

### Q2：延迟很高怎么办？

**解决方法**：
1. 调整 `minCache` 和 `maxCache` 参数
```javascript
config: {
  minCache: 0.5,  // 减小最小缓冲
  maxCache: 1,    // 减小最大缓冲
  mode: 'RTC'     // 使用RTC模式（延迟更低）
}
```

2. 使用低延迟直播方案（WebRTC）
3. 优化推流端设置（降低GOP）

### Q3：如何实现多清晰度切换？

**方法**：
```javascript
// 提供多个流地址
export default {
  streams: {
    high: 'https://play.yourdomain.com/live/stream_1080p.m3u8',
    medium: 'https://play.yourdomain.com/live/stream_720p.m3u8',
    low: 'https://play.yourdomain.com/live/stream_480p.m3u8'
  },
  
  // 默认使用中等清晰度
  liveStreamUrl: 'https://play.yourdomain.com/live/stream_720p.m3u8'
}
```

### Q4：如何知道直播是否在线？

**方法1：后端定时检测**
```javascript
// 后端定期检测流状态
setInterval(async () => {
  const isOnline = await checkStreamOnline(streamUrl);
  if (isOnline) {
    // 广播直播在线状态
    broadcast('liveStatus', { isLive: true });
  }
}, 30000);
```

**方法2：监听播放器事件**
```javascript
// 小程序中监听
<live-player 
  @statechange="handleStateChange"
  @error="handleError"
/>
```
