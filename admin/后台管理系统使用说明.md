# 后台管理系统使用说明

## 📘 功能概述

本后台管理系统可以：
1. ✅ 实时查看小程序数据（用户、投票、AI内容）
2. ✅ 控制直播开始/停止
3. ✅ 手动调整投票数据
4. ✅ 控制AI识别开启/关闭
5. ✅ 通过WebSocket实时同步数据到小程序

---

## 🔧 配置服务器地址

### 修改服务器配置

打开 `admin/admin.js` 文件，找到第3-12行的服务器配置：

```javascript
const SERVER_CONFIG = {
	// 本地开发时使用
	LOCAL_URL: 'http://localhost:8000',
	// 真实服务器地址（后端开发完成后使用）
	REAL_URL: 'http://192.168.31.189:8000',
	// 当前使用的地址（修改这里切换服务器）
	get BASE_URL() {
		return this.REAL_URL; // 切换到 REAL_URL 使用真实服务器
	}
};
```

**切换方法**：
- **使用本地服务器**：将 `return this.REAL_URL;` 改为 `return this.LOCAL_URL;`
- **使用真实服务器**：将 `return this.REAL_URL;` (默认配置)

## 🎯 核心功能使用

### 1. 数据概览页面

**功能**：
- 查看总用户数、在线用户数
- 查看直播状态
- 查看总投票数
- 实时数据通过WebSocket自动更新

**操作**：
- 点击侧边栏「📊 数据概览」进入
- 页面会自动调用 `GET /api/admin/dashboard` 接口
- WebSocket连接成功后会实时同步数据

---

### 2. 直播控制

#### 开始直播

**操作步骤**：
1. 点击顶部栏的「开始直播」按钮
2. 如果有多个直播流，选择一个直播流
3. 选择是否自动启动AI识别
4. 点击确认

**接口调用**：
```javascript
// 调用新接口
await startLive(streamId, autoStartAI);
// 等价于
POST /api/admin/live/start
{
  "streamId": "stream_uuid_123",
  "autoStartAI": true,
  "notifyUsers": true
}
```

**效果**：
- 后台管理系统显示「🟢 直播中」
- 小程序通过WebSocket收到`live-started`消息并自动开始播放
- 如果选择了自动启动AI，AI识别也会同时启动

#### 停止直播

**操作步骤**：
1. 点击顶部栏的「停止直播」按钮
2. 确认操作

**接口调用**：
```javascript
await stopLive();
// 等价于
POST /api/admin/live/stop
```

**效果**：
- 后台管理系统显示「⚪ 未开播」
- 小程序通过WebSocket收到`live-stopped`消息并停止播放
- 保存本场直播的统计数据

---

### 3. 投票数据控制

**页面**：点击侧边栏「🗳️ 票数管理」

#### 功能一：手动设置票数

**操作**：
1. 在「手动设置票数」区域
2. 输入正方票数和反方票数
3. 输入调整原因（可选）
4. 点击「设置票数」按钮

**接口调用**：
```javascript
await updateVotes('set', leftVotes, rightVotes, reason);
// 等价于
POST /api/admin/live/update-votes
{
  "action": "set",
  "leftVotes": 30000,
  "rightVotes": 20000,
  "reason": "数据校正",
  "notifyUsers": true
}
```

**效果**：
- 票数立即更新
- 小程序通过WebSocket收到`votes-updated`消息
- 对抗条实时更新显示

#### 功能二：增加票数

**操作**：
1. 在「增加票数」区域
2. 输入要增加的票数
3. 点击「增加票数」按钮

**接口调用**：
```javascript
await updateVotes('add', leftVotesIncrease, rightVotesIncrease);
```

#### 功能三：重置票数

**操作**：
1. 在「重置票数」区域
2. 输入重置后的票数（通常是0）
3. 点击「重置票数」按钮

**接口调用**：
```javascript
await resetVotes(0, 0);
// 等价于
POST /api/admin/live/reset-votes
{
  "resetTo": {
    "leftVotes": 0,
    "rightVotes": 0
  },
  "saveBackup": true,
  "notifyUsers": true
}
```

**效果**：
- 当前数据会被备份
- 票数重置为指定值
- 小程序实时同步

### 4. AI识别控制

**页面**：点击侧边栏「🤖 AI 内容管理」

#### 启动AI识别

**操作**：
1. 点击「启动AI识别」按钮
2. （可选）配置AI设置：
   - 模式：realtime（实时）/ interval（定时）
   - 敏感度：low / medium / high
   - 最小置信度：0-1

**接口调用**：
```javascript
await startAI({
  mode: 'realtime',
  sensitivity: 'high',
  minConfidence: 0.7
});
// 等价于
POST /api/admin/ai/start
{
  "settings": {
    "mode": "realtime",
    "interval": 5000,
    "sensitivity": "high",
    "minConfidence": 0.7
  },
  "notifyUsers": true
}
```

**效果**：
- AI开始识别直播内容
- 生成的AI内容会实时推送到小程序
- 小程序显示AI总结的观点

#### 停止AI识别

**操作**：
1. 点击「停止AI识别」按钮

**接口调用**：
```javascript
await stopAI();
// 等价于
POST /api/admin/ai/stop
```

#### 暂停/恢复AI识别

**操作**：
1. 点击「暂停AI识别」按钮（暂时停止，不结束会话）
2. 点击「恢复AI识别」按钮（继续识别）

**接口调用**：
```javascript
await toggleAI('pause');  // 暂停
await toggleAI('resume'); // 恢复
```

#### 删除AI内容

**操作**：
1. 在AI内容列表中找到要删除的内容
2. 点击「删除」按钮
3. 输入删除原因（可选）
4. 确认删除

**接口调用**：
```javascript
await deleteAIContent(contentId, reason);
// 等价于
DELETE /api/admin/ai/content/{contentId}
```

**效果**：
- 内容从列表中移除
- 小程序通过WebSocket收到`ai-content-deleted`消息
- 小程序同步删除该内容

## 🔌 WebSocket实时推送

### 连接状态指示

顶部栏右侧有连接状态指示器：
- **🟢 绿点**：WebSocket已连接，实时同步正常
- **🔴 红点**：WebSocket断开，5秒后自动重连

### 推送消息类型

后台管理系统会自动处理以下WebSocket消息：

| 消息类型 | 触发场景 | 自动处理 |
|---------|---------|---------|
| `state` | 首次连接 | 同步初始状态 |
| `live-started` | 直播开始 | 更新直播状态为「直播中」 |
| `live-stopped` | 直播停止 | 更新直播状态为「未开播」 |
| `votes-updated` | 票数变更 | 更新票数显示 |
| `ai-started` | AI启动 | 更新AI状态 |
| `ai-stopped` | AI停止 | 更新AI状态 |
| `ai-status-changed` | AI暂停/恢复 | 更新AI状态 |
| `ai-content-added` | AI生成内容 | 刷新内容列表 |
| `ai-content-deleted` | AI内容删除 | 刷新内容列表 |

## 📱 与小程序的联动

### 1. 直播控制联动

**场景一：开始直播**
1. 后台点击「开始直播」→ 调用 `POST /api/admin/live/start`
2. 服务器通过WebSocket推送 `live-started` 消息
3. 小程序收到消息 → 自动开始播放直播
4. 后台和小程序同步显示「直播中」状态

**场景二：停止直播**
1. 后台点击「停止直播」→ 调用 `POST /api/admin/live/stop`
2. 服务器通过WebSocket推送 `live-stopped` 消息
3. 小程序收到消息 → 停止播放
4. 后台和小程序同步显示「未开播」状态

### 2. 投票数据联动

**场景：手动调整票数**
1. 后台输入票数并点击「设置票数」→ 调用 `POST /api/admin/live/update-votes`
2. 服务器通过WebSocket推送 `votes-updated` 消息
3. 小程序收到消息 → 对抗条实时动画更新
4. 用户看到票数平滑变化

### 3. AI识别联动

**场景：启动AI识别**
1. 后台点击「启动AI识别」→ 调用 `POST /api/admin/ai/start`
2. 服务器通过WebSocket推送 `ai-started` 消息
3. 小程序收到消息 → 开始显示「AI识别中」状态
4. AI生成内容后 → 推送 `ai-content-added` 消息
5. 小程序实时显示新的AI总结

## ⚠️ 常见问题

### 1. WebSocket连接失败

**问题**：状态指示器一直是红色

**原因**：
- 服务器未启动
- 服务器地址配置错误
- 网络问题

**解决**：
1. 确认服务器已启动：`node server.js`
2. 检查 `admin.js` 中的 `SERVER_CONFIG.REAL_URL` 是否正确
3. 检查防火墙设置

### 2. 接口调用失败

**问题**：点击按钮后提示"操作失败"

**原因**：
- 后端接口未开发
- 服务器地址错误
- 跨域问题

**解决**：
1. 确认后端同事已开发对应接口
2. 检查服务器配置
3. 使用Node.js服务器启动（而不是直接打开HTML）

### 3. 数据不同步

**问题**：小程序和后台数据不一致

**原因**：
- WebSocket断开
- 小程序未正确处理推送消息

**解决**：
1. 检查WebSocket连接状态
2. 刷新页面重新连接
3. 检查小程序的WebSocket消息处理逻辑

## 📚 扩展功能

### 添加新按钮

如果需要添加新的控制按钮，可以：

1. 在 `admin/index.html` 中添加按钮
2. 在 `admin/admin.js` 中添加事件监听
3. 调用对应的API函数

**示例**：
```javascript
// 添加按钮点击事件
document.getElementById('my-button').addEventListener('click', async () => {
  const result = await startLive();
  if (result) {
    console.log('操作成功:', result);
  }
});
```

### 自定义通知

默认使用 `alert()` 显示通知，可以修改 `showNotification` 函数来美化：

```javascript
function showNotification(message, type = 'info') {
  // 使用Toast库或自定义UI
  // 例如：Toastify.toast({ text: message, type });
  alert(message); // 当前实现
}
```
